# Capstone Business Analysis Project - README

## What evaluators expect (exact deliverables)
1. **Problem definition & Business Objective** - Short description of the business problem you choose and why it matters.
2. **Data source & Description** - A CSV file with the sales data and a data dictionary describing columns.
3. **Data cleaning & preprocessing steps** - Document missing values, transformations, feature engineering.
4. **Exploratory Data Analysis (EDA)** - Summary statistics, trend analysis (monthly revenue), and key takeaways.
5. **Visualizations** - Time series, box plots, violin plots, heatmap, and multiple-subplot figures demonstrating different views.
6. **Customer segmentation** - RFM analysis and KMeans clustering with segment descriptions.
7. **Product & Category performance analysis** - Top products, revenue share, and recommendations.
8. **Actionable Recommendations & Implementation Plan** - Business-facing next steps with concrete actions and expected impact.
9. **Project documentation & reproducibility** - Notebook, saved datasets (aggregates), README, and a short presentation.
10. **Professional presentation** - A PPTX summarizing findings for non-technical stakeholders.

## Files included
- analysis.ipynb : Jupyter notebook with full analysis
- sales_data.csv : Synthetic sample dataset
- rfm_segments.csv, monthly_summary.csv : Derived files
- presentation.pptx : Slide deck summarizing the analysis
- requirements.txt : Packages needed to run the notebook
- README.md : This file

## How to run
1. Create a conda or venv environment, install requirements from `requirements.txt`.
2. Open `analysis.ipynb` and run all cells.
3. Replace `sales_data.csv` with your real CSV (same columns) to repeat the analysis on real data.
